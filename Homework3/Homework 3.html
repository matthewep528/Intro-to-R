Exploratory data analysis

Purpose: This assignment aims to assess your ability to perform exploratory data analysis using the skills you learned through module 10. You will demonstrate your ability to explore missing data patterns and produce numerical and graphical data summaries. To receive full credit, you must upload the script you used to create and label the data, the markdown file you used for exploratory data analysis, and your rendered HTML. Also, you should use good programming practices (i.e., documentation, comments, spacing, indentation). See the rubric for details on the scoring breakdown.

Data frame:   diet_data-2.csv

Codebook: Diet_Data_Codebook.xls

 

Part 1.

  1. Create a R Script to import the diet_data.csv file (this is the same csv file used for homework 2)
  2. Examine the structure and dimensions to ensure the data frame was uploaded correctly (confirm there are 12 variables, 1000 observations)
  3. Create a data frame of only adults (>20 years of age)
  4. Create a data frame of only non-adults (<20 years of age)
  5. Label variables and values in the adult data frame using the Diet_Data_Codebook

 

Part 2.

  1. Create a R Markdown file to inspect the adult data frame
  2. What percentage of the data frame has complete data (i.e., non-missing)
  3. Which variable has the highest amount of missing data
  4. Which variable has the lowest amount of missing data
  5. Provide summary statistics for each variable
  6. Based on the summary statistics, which continuous variables are normally distributed
  7. Which continuous variables are skewed
  8. Do the range of values look for continuous variables look reasonable
  9. For the categorical variables, do any variable have small counts on any level
  10. In addition to producing univariate summary statistics, produce summary statistics by the primary exposure variable, 10-hour food intake window (yes vs no)
  11. Create graphical summaries for each variable, univariate and by the primary exposure variable

Upload your R scripts AND markdown generated HTML file

Use good programming practices (i.e., documentation, comments, spacing, indentation)
